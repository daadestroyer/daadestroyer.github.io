Smart Contact Manager

SCM

About Project

  • Project Type : Web Application
  • Technologies Used : Spring Boot , Spring Data JPA , Thymeleaf , Spring Security , Hibernate Validator , MySql Workbench 8.0
  • Team Size : Only Me
SOURCE CODE

Features
  • User can register himself in smart contact manager , all the validation are done using Hibernate Validator before registering User
  • User can do login , Custmized spring security login page and protect all URL's (start with /user) from unregistred user means some pages are secure from unregistered user
  • Use Bencryptor to encrypt user password and store encrypted password in DB
  • User can add contacts with her contact image in DB
  • User can see its contact list in /view-contact section
  • User can search her contact from Search Box
  • Used some Javascript along with Ajax and by using findByNameContainingAndUser() method
  • User can see her contact's profile
  • User can delete contacts
  • User can update contacts also
  • If user manually entered some id to delete some contact which doesn't exist in DB then no user found of this ID type of error will come ex. https://localhost:8080/User/delete-contact/11 If contact with id 11 doesn't exist in DB then error like no user found of this id will come
  • If user A trying to delete user B contact whoose ID is suppose 10 ofcourse that user is not a contact of user A still if User A trying to delete that contact by manually putting it id in URL then error like no user found of this id will come
  • Edit contact funcitonality is implement under GET mapping if user edit contact by copying edit contact link then in that case Method Not Allow here error 405 will come , means user can edit contact by clicking only on edit button link
  • If User manually trying to open some page which doesn't exit then Page Not Found error 404 come ex. https://localhost:8080/User/abcdef ex. abcdef page doesnt't exist
  • If any other type of error come then error 500 Internal Server Error come
  • Implement this Centralized Excpetion Handling in single page called error.html
  • User can change its password but first user have to enter its old password then he can change password
  • Implement forgot password by getting OTP in it email , after successfull verification of OTP user can change it password
